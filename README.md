# hexlet-go-crawler

Парсер сайтов на языке Go для анализа структуры веб-ресурсов. Поддерживает многопоточный обход, ограничение скорости, повторные запросы при ошибках, проверку битых ссылок и внешних ассетов.

## Быстрый старт

```bash
# Клонировать репозиторий
git clone <repo-url>
cd hexlet-go-crawler

# Установить зависимости
go mod tidy

# Скомпилировать бинарник
make build

# Запустить краулер
make run URL=https://example.com
```

## Параметры запуска
### Основные флаги
|    Флаг      | Краткий | Значение по умолчанию |           Описание                                        |
|--------------|---------|-----------------------|-----------------------------------------------------------|
| --depth      | -d      | 10                    | Максимальная глубина обхода ссылок                        |
| --retries    | —       | 1                     | Количество повторных попыток при временных ошибках        |
| --delay      | —       | 0                     | Фиксированная пауза между запросами (например, 200ms, 1s) |
| --timeout    | —       | 15s                   | Таймаут каждого HTTP-запроса                              |
| --rps        | —       | 0                     | Лимит запросов в секунду (приоритетнее --delay)           |
| --user-agent | -ua     | hexlet-go-crawler/1.0 | Заголовок User-Agent для запросов                         |
| --workers    | -w      | 4                     | Количество параллельных воркеров                          |
| --pretty     | -p      | false                 | Красивое форматирование JSON-вывода                       |

#### Примеры использования
```bash
# Базовый запуск
make run URL=https://example.com

# Обход с кастомными параметрами
go run ./cmd/hexlet-go-crawler \
  --depth 2 \
  --timeout 30s \
  --workers 8 \
  --user-agent "MyBot/1.0" \
  --pretty \
  https://example.com

# Просмотр справки
go run ./cmd/hexlet-go-crawler --help
```

## Глубина обхода
Параметр --depth определяет максимальное количество переходов от стартовой страницы:

| Значение | Поведение                                                 |
|----------|-----------------------------------------------------------|
| 0        | Обрабатывается только стартовая страница                  |
| 1        | Стартовая страница + ссылки первого уровня (по умолчанию) |
| N        | Обход продолжается до N-го уровня вложенности             |

### Ограничения обхода

    Домен: краулер следует только по ссылкам внутри исходного домена. Внешние ссылки проверяются на доступность, но не добавляются в очередь обхода.
    Дедупликация: дубликаты URL автоматически отсекаются — каждая страница появляется в отчёте ровно один раз.
    Отмена: при прерывании (Ctrl+C или context cancellation) краулер завершает текущие запросы и возвращает валидный JSON с уже собранными данными.

```bash
# Пример: обход до 3-го уровня
make run URL=https://example.com DEPTH=3
```

## Ограничение скорости (--delay / --rps)
Краулер позволяет глобально ограничивать частоту HTTP-запросов, чтобы не перегружать целевой сайт.

|     Флаг      | Описание                              | Приоритет                     |
|---------------|---------------------------------------|-------------------------------|
| --delay=200ms | Фиксированная пауза между запросами   | Низкий                        |
| --rps=5       | Целевое количество запросов в секунду | Высокий (перекрывает --delay) |

Если оба параметра не указаны, запросы отправляются максимально быстро (ограничено только ```--timeout``` и ```--workers```).

Примеры

```bash
# 5 запросов в секунду (интервал ~200ms)
make run URL=https://example.com RPS=5

# Фиксированная задержка 1 секунда
make run URL=https://example.com DELAY=1s
```

## Повторные запросы (--retries)

Краулер автоматически повторяет запросы при временных ошибках сервера или сети.
### Поведение

| Ситуация                                   |     Действие                      |
|--------------------------------------------|-----------------------------------|
| Сетевой сбой (connection refused, timeout) | Повторить запрос                  |
| 429 Too Many Requests                      | Повторить запрос                  |
| 5xx Server Error                           | Повторить запрос                  |
| 2xx, 3xx, 4xx (кроме 429)                  | Считать окончательным результатом |


### Параметры

* Задержка: используется ```--delay``` (по умолчанию 200ms) с линейным увеличением (delay × номер_попытки).
* Лимит: ```--retries=N``` задаёт количество дополнительных попыток (всего N+1 запросов к серверу).
* Отмена: при отмене контекста ожидание между попытками немедленно прерывается.

```bash
# 3 попытки (1 основная + 2 повтора) с задержкой 500ms
make run URL=https://example.com RETRIES=2 DELAY=500ms
```

## Проверка ассетов
Краулер автоматически анализирует внешние ресурсы страницы:

| Тег                       | Атрибут | Тип ассета |
|---------------------------|---------|------------|
| `<img>`                   | src     | image      |
| `<script>`                | src     | script     |
| `<link rel="stylesheet">` | href    | style      |

### Поля отчёта для ассета

|    Поле     | Тип    |                     Описание                              |
|-------------|--------|-----------------------------------------------------------|
| url         | string | Абсолютный URL ресурса                                    |
| type        | string | Тип: image, script, style, other                          |
| status_code | int    | HTTP-статус ответа                                        |
| size_bytes  | int64  | Размер в байтах (из Content-Length или фактического тела) |
| error       | string | Сообщение об ошибке (пустая строка при успехе)            |


### Кэширование
Один и тот же ассет, встреченный на разных страницах, проверяется только один раз. Результаты переиспользуются, лишние запросы к серверу не выполняются.

```bash
# Пример: просмотр ассетов первой страницы
make run URL=https://example.com | jq '.pages[0].assets'
```

## Формат отчёта
Краулер выводит результат в строгом формате JSON. Все ключи обязательны, даже если их значение пустое. Временные поля используют формат RFC3339 (ISO8601).
### Корневой объект

|     Поле     |   Тип  | Описание                                |
|--------------|--------|-----------------------------------------|
| root_url     | string | Исходный URL, с которого начался обход  |
| depth        | int    | Максимальная глубина обхода из настроек |
| generated_at | string | Время генерации отчёта (RFC3339)        |
| pages        | array  | Массив объектов страниц                 |

### Объект страницы (pages[])

|   Поле        | Тип    | Описание                                      | 
|---------------|--------|-----------------------------------------------|
| url           | string | Абсолютный URL страницы                       |
| depth         | int    | Глубина страницы относительно корня           |
| http_status   | int    | HTTP-статус ответа (0 при сетевой ошибке)     |
| status        | string | "ok", "error" или "skipped"                   |
| error         | string | Сообщение об ошибке (пустая строка, если нет) |
| broken_links  | array  | Массив битых ссылок на странице               |
| discovered_at | string | Время обнаружения страницы (RFC3339)          |
| seo           | object | SEO-метрики (см. ниже)                        |
| assets        | array  | Массив ассетов (см. выше)                     | 

### Объект seo

| Поле            | Тип    | Описание                                   |
|-----------------|--------|--------------------------------------------|
| has_title       | bool   | Наличие непустого `<title>`                |
| title           | string | Текст заголовка (пустой, если не найден)   |
| has_description | bool   | Наличие <meta name="description">          |
| description     | string | Текст описания (пустой, если не найден)    |
| has_h1          | bool   | Наличие непустого `<h1>`                   |

### Объект broken_links[]

| Поле        | Тип    | Описание                           |
|-------------|--------|------------------------------------|
| url         | string | URL битой ссылки                   |
| status_code | int    | HTTP-статус (0 при сетевой ошибке) |
| error       | string | Текст ошибки                       |

## Форматирование вывода
Флаг ```--pretty``` (или IndentJSON: true в API) добавляет отступы и переносы строк для читаемости. Содержимое и порядок ключей при этом не меняются.

### Полный пример отчёта

```json
{
  "root_url": "https://example.com",
  "depth": 1,
  "generated_at": "2024-06-01T12:34:56Z",
  "pages": [
    {
      "url": "https://example.com",
      "depth": 0,
      "http_status": 200,
      "status": "ok",
      "error": "",
      "broken_links": [
        {
          "url": "https://example.com/missing",
          "status_code": 404,
          "error": "Not Found"
        }
      ],
      "discovered_at": "2024-06-01T12:34:56Z",
      "seo": {
        "has_title": true,
        "title": "Example title",
        "has_description": true,
        "description": "Example description",
        "has_h1": true
      },
      "assets": [
        {
          "url": "https://example.com/static/logo.png",
          "type": "image",
          "status_code": 200,
          "size_bytes": 12345,
          "error": ""
        }
      ]
    }
  ]
}
```

## Архитектура
### Компоненты 
| Пакет                 | Ответственность                                                                 |
|-----------------------|---------------------------------------------------------------------------------|
| cmd/hexlet-go-crawler | CLI-интерфейс: парсинг флагов, запуск Analyze, вывод результата                 | 
| crawler               | Основная логика: обход, парсинг, проверка ссылок и ассетов, формирование отчёта |

### Тестируемость

* http.Client инжектируется через Options, что позволяет подменять его в тестах.
* Все зависимости передаются через интерфейсы или параметры функций.
* Тесты используют httptest.Server для изоляции от внешних ресурсов.
